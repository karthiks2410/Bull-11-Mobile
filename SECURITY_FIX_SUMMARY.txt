BULL-11 SECURITY FIX - DIRECT URL NAVIGATION PROTECTION
================================================================================

ISSUE FIXED:
------------
Users could bypass authentication by directly typing URLs like:
  - http://localhost:5050/(tabs)
  - http://localhost:5050/(tabs)/profile
  - http://localhost:5050/(tabs)/admin
  - http://localhost:5050/(admin)/dashboard

This allowed unauthorized access to protected content without login.

SOLUTION:
---------
Implemented AuthGuard and AdminGuard components that intercept ALL route 
navigation and verify authentication/authorization before rendering content.

KEY COMPONENTS:
---------------
1. AuthGuard (/src/presentation/guards/AuthGuard.tsx)
   - Wraps entire app at root layout level
   - Checks isAuthenticated state
   - Monitors route segments with useSegments()
   - Redirects unauthenticated users to /auth/login
   - Prevents content flash with loading states
   - Updates session activity on navigation

2. AdminGuard (/src/presentation/guards/AdminGuard.tsx)
   - Wraps admin routes
   - Extends AuthGuard with requireAdmin=true
   - Blocks non-admin users from admin routes

INTEGRATION POINTS:
-------------------
✓ Root Layout (app/_layout.tsx) - Wrapped with AuthGuard
✓ Admin Layout (app/(admin)/_layout.tsx) - Wrapped with AdminGuard
✓ Tabs Layout (app/(tabs)/_layout.tsx) - Redundant guard removed

SECURITY FEATURES:
------------------
✓ Route segment monitoring prevents redirect loops
✓ Loading states prevent content flash
✓ Token validation on every route access
✓ Session activity tracking
✓ Console logging for debugging
✓ Race condition protection
✓ Works with existing session timeout system

HOW IT WORKS:
-------------
Before (Vulnerable):
  User types URL → Route loads → Content renders → No auth check

After (Protected):
  User types URL → AuthGuard intercepts → Check auth → 
  If not authenticated: Show spinner → Redirect to login
  If authenticated: Render content
